Elliptic Curve Visualization

Why can't you reverse it?

Given public key Q = k * G (where k is the private key and G is the generator point), finding k requires solving the Elliptic Curve Discrete Logarithm Problem. With a 256-bit key, there are ~10^77 possible values - more than atoms in the observable universe. Point multiplication is a one-way trapdoor function: easy to compute forward, computationally infeasible to reverse.

Sign a Message

1Hash the message: z = SHA256(msg)
2Pick random nonce k_e
3Compute R = k_e * G, then r = R.x mod n
4Compute s = k_e^(-1) * (z + r * privKey) mod n
5Signature = (r, s)

Verify Signature

1Compute z = SHA256(msg)
2Compute u1 = z * s^(-1) mod n
3Compute u2 = r * s^(-1) mod n
4Compute P = u1*G + u2*Q
5Check: P.x mod n == r?

Tamper Demonstration

Change the message after signing to see signature verification fail.

Why does tampering fail?

The signature (r, s) is mathematically bound to the exact hash of the original message. Even changing a single character produces a completely different hash (avalanche effect), making the signature invalid. An attacker cannot forge a valid signature without knowing the private key.
